Executive Summary
ChainBois is a competitive third-person shooter built around the concept of Playable Identity.
In traditional games, progression lives inside an account. Players grind for months or years, but that progress cannot be transferred, traded, or owned.
ChainBois changes this model by tying progression directly to NFTs. Each ChainBoi NFT records gameplay performance such as wins, kills, and tournament results. These statistics increase the value, rarity, and competitive status of the character.
Instead of static collectibles, ChainBois NFTs become evolving competitive identities.
The project already has a working MVP deployed on Avalanche Fuji testnet including:
• Deployed smart contracts
• A working multiplayer game client (PC and Android)
• Backend infrastructure supporting tournaments and tokenomics
• NFT progression mechanics
• Anti-cheat score validation
• A full player onboarding system
The Avalanche grant would accelerate the transition from MVP to scaled competitive ecosystem by supporting infrastructure scaling, player acquisition, and tournament expansion.
Problem
Web3 gaming has struggled with long-term player retention.
Most blockchain games rely on speculative token economies instead of engaging gameplay. Assets are often static collectibles that do not evolve based on player performance.
Common problems include:
• NFTs with no gameplay relevance
• Progression tied to centralized accounts
• Token emissions that collapse under inflation
• Poor onboarding experiences for non-crypto players
As a result, most Web3 games experience rapid user churn after the initial launch hype.
Meanwhile, competitive multiplayer games in Web2 continue to dominate global engagement. Titles like Call of Duty, Apex Legends, and Valorant demonstrate that players are highly motivated by skill-based competition, rankings, and prestige.
The missing piece in Web3 gaming is a system where gameplay skill directly affects the value and identity of digital assets.
Solution
ChainBois introduces Playable Identity.
Playable Identity means that the character itself becomes the record of a player's achievements.
Instead of accounts holding progress, the NFT character holds:
• Wins
• Kills
• Tournament placements
• Level progression
• Rank status
As a player improves, their ChainBoi evolves. The NFT's metadata updates dynamically and reflects that performance.
This system creates three key advantages:

1. Skill-based asset value
   Better players create more valuable characters.
2. Tradable progression
   Players can sell upgraded identities rather than starting from zero.
3. Long-term engagement
   Characters become personal competitive assets with history and prestige.

   Technical Architecture
   ChainBois is built using a modular architecture that separates gameplay systems from blockchain infrastructure.
   Game Engine
   Unity handles gameplay, physics, rendering, and multiplayer interaction.
   Backend Layer
   A Node.js / Express API manages game state synchronization, tournaments, NFT progression logic, and token economy automation.
   Database Layer
   MongoDB stores persistent platform data while Firebase Realtime Database acts as a bridge between the game client and backend.
   Blockchain Layer
   Avalanche C‑Chain smart contracts manage NFT ownership, token balances, and asset transfers.
   Smart Contracts
   • BattleToken (ERC20 capped token)
   • ChainBoisNFT (ERC721 characters)
   • WeaponNFT (ERC721 weapons)
   This architecture allows the game to remain responsive while still benefiting from blockchain transparency and ownership.
